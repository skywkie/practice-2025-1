# Отчет по проекту "Build Your Own React"
Источник: [Build-your-own-react](pomb.us/build-your-own-react)
Автор: Rodrigo Pombo
Дата публикации: 13 ноября 2019 г.
Цель проекта: Пошаговое создание собственной библиотеки, копирующей архитектуру React 16.8, с нуля, с поддержкой хуков и отказом от классов.

## Введение и Цели
Данный проект представляет собой образовательное руководство, цель которого — воссоздать базовую функциональность React, следуя архитектуре реального кода React, но без оптимизаций и второстепенных функций. Версия React 16.8 позволяет сосредоточиться на современном подходе с использованием хуков. Проект реализует библиотеку под названием Didact.

Процесс разбит на восемь шагов, от самых базовых функций до полноценной поддержки хуков.

## Пошаговая Реализация
### Шаг 0: Обзор Основ (Review)
Перед началом разработки проводится ревизия базовых концепций: JSX, элементов React и DOM-элементов. На примере приложения из трех строк показывается, как JSX-код вида:

jsx
const container = document.getElementById("root")
ReactDOM.render(element, container)
...транспилируется Babel в вызов React.createElement и затем в обычный объект элемента с полями type и props. Затем вручную показано, как создать соответствующий DOM-узел, установить его свойства и добавить в контейнер, заменяя вызов ReactDOM.render.

### Шаг I: Функция createElement
Создается собственная функция createElement, которая формирует объект элемента.

Задача: Преобразовать JSX в объекты, понятные библиотеке.

Реализация: Функция принимает type, props и дочерние элементы (...children). Возвращает объект { type, props: { ...props, children } }.

Особенности:

Дочерние элементы всегда формируются в массив для упрощения кода.

Примитивные значения (строки, числа) оборачиваются в объект со специальным типом TEXT_ELEMENT.

Настройка JSX: Для указания Babel использовать собственную функцию createElement вместо React.createElement, используется комментарий /** @jsx Didact.createElement */.

### Шаг II: Функция render
Создается начальная версия функции render для добавления элементов в DOM.

Задача: Рекурсивно создать все DOM-узлы и добавить их в контейнер.

Процесс:

Для каждого элемента создается DOM-узел (document.createElement или document.createTextNode для TEXT_ELEMENT).
Узлу присваиваются пропсы (атрибуты, обработчики).
Дочерние элементы обрабатываются рекурсивно.
Готовый узел добавляется к родительскому.
### Шаг III: Конкурентный Режим (Concurrent Mode)
Проблема рекурсивного рендеринга — блокировка главного потока. Решение — разбиение работы на маленькие юниты и использование requestIdleCallback.

Цель: Позволить браузеру прерывать рендеринг для обработки высокоприоритетных задач (ввод, анимации).

Реализация: Создается цикл workLoop, который вызывает requestIdleCallback для планирования работы. Вводится глобальная переменная nextUnitOfWork, которая указывает на следующий юнит работы (волокно).

### Шаг IV: Волокна (Fibers)
Вводится структура данных — дерево волокон (fiber tree). Волокно — это юнит работы, связанный с каждым элементом.

Структура Волокна: Каждое волокно хранит ссылки на своего родителя (parent), первого потомка (child) и следующего соседа (sibling). Это облегчает поиск следующего юнита работы.

Алгоритм обхода в performUnitOfWork:

Добавить элемент в DOM (позже будет перенесено).
Создать волокна для дочерних элементов.
Вернуть следующий юнит работы: сначала child, если его нет — sibling, если и его нет — sibling родителя ("дядя"), и так далее вверх по дереву до корня.
### Шаг V: Фазы Render и Commit
Проблема поэтапного добавления узлов в DOM во время работы цикла — пользователь может увидеть неполный UI. Решение — разделение процесса на две фазы.

Фаза Render (Отрисовка): Волокна только обрабатываются, но DOM не изменяется. Результатом является полностью построенное дерево волокон (wipRoot).

Фаза Commit (Фиксация): Когда вся работа завершена (!nextUnitOfWork), вызывается commitRoot для рекурсивного и единовременного применения изменений к DOM. Сохраняется ссылка на зафиксированное дерево (currentRoot).

### Шаг VI: Согласование (Reconciliation)
Добавляется логика обновления и удаления узлов. Необходимо сравнивать новые элементы со старыми волокнами из последней фиксации.

Механизм: У каждого волокна появляется свойство alternate, ссылающееся на его предыдущую версию.

Функция reconcileChildren: Сравнивает старые волокна (oldFiber) и новые элементы (element) по типу:

Типы совпадают: Создается новое волокно, сохраняющее старый DOM-узел, с пропсами из нового элемента и тегом эффекта UPDATE.

Типы разные (новый элемент): Создается новое волокно с тегом PLACEMENT.

Типы разные (старое волокно): Старое волокно помечается тегом DELETION и добавляется в массив deletions.

Применение в commitWork: Функция обрабатывает теги эффектов: для PLACEMENT — добавляет узел, для DELETION — удаляет, для UPDATE — обновляет изменившиеся пропсы, включая обработчики событий.

### Шаг VII: Функциональные Компоненты
Добавляется поддержка компонентов-функций.

Отличия:

Волокно функционального компонента не имеет собственного DOM-узла.

Дочерние элементы получаются не из props, а путем вызова функции (fiber.type(fiber.props)).

Изменения:

В performUnitOfWork происходит ветвление: для функций вызывается updateFunctionComponent, для обычных элементов — updateHostComponent.

В функции commitWork для поиска родительского DOM-узла или удаляемого дочернего DOM-узла нужно подниматься по цепочке волокон вверх/вниз, пропуская компоненты без DOM-узлов.

### Шаг VIII: Хуки (Hooks)
Реализуется поддержка хука useState.

Глобальные переменные: Перед вызовом функционального компонента устанавливаются текущее волокно (wipFiber) и его массив hooks с индексом hookIndex.

Хранение состояния: Состояние и очередь действий хранятся в каждом хуке волокна.

Логика useState(initial):

Проверяется, есть ли старый хук (alternate). Если есть, состояние копируется; если нет, инициализируется.
Создается функция setState(action), которая добавляет action в очередь хука и запускает новый цикл рендеринга с корневого волокна, устанавливая новую единицу работы.
При следующем рендере все действия из очереди применяются к состоянию для получения актуального значения.
Функция возвращает текущее состояние и setState.

## Эпилог и Отличия от Реального React
Проект завершен, создана работающая минималистичная версия React. Для облегчения дальнейшего изучения исходного кода React используются те же имена переменных и функций.

Основные упрощения по сравнению с реальным React:

Обход Дерева: React пропускает неизмененные поддеревья с помощью эвристик.

Фаза Commit: React работает только со списком волокон с эффектами.

Переиспользование Объектов: React переиспользует волокна из предыдущих деревьев.

Приоритеты Обновлений: React использует временные метки истечения для определения приоритета обновлений.

Возможные для самостоятельного добавления функции:

Обработка стилей через объект в пропсах.

Поддержка useEffect.

Согласование по ключам (key).

Сведение массивов-детей.

## Заключение
Проект "Build Your Own React" — это ценный образовательный ресурс, который на практике демонстрирует внутреннее устройство React: от создания элементов и рендеринга до волокон, согласования и хуков. Пошаговый подход с чистого листа позволяет глубоко понять мотивацию и механизмы работы библиотеки, опуская детали оптимизаций. Полученный код, хотя и не предназначен для production, концептуально очень близок к реальному React.
